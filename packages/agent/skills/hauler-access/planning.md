# Plan scheduled service access

1. Separate the desired collection outcome from hard identity, hatch, time-window, and final-state limits.
2. Read the assigned hauler tag, service-hatch capability, resident-hatch policy, existing routines, and protected versions.
3. Keep the residential hatch locked for every proposed action.
4. Require the service hatch to finish locked and include evidence for that final state.
5. Replace overlapping hauler access instead of creating a second active routine.

The model cannot widen the window, change the authorized identity, or grant itself access authority.
